# 文章阅读增强

文章详情页已提供以下阅读功能：

- 基于正文区域的阅读进度条。
- 目录平滑定位、当前章节自动高亮。
- 代码块语言标签、行号、复制与折叠。
- 正文图片灯箱，支持键盘 `Esc` 关闭、上一张/下一张和查看原图。
- 文章底部的上一篇、下一篇、相关路径与同主题文章。
- `/map` 文章知识图谱：以显式关联、系列与共同标签展示文章网络。

## 变更记录

在文章 frontmatter 中可选填写 `changelog`。按数组顺序展示，建议将最新记录放在最前。

```yaml
updatedDate: '2026-07-20'
changelog:
  - date: '2026-07-20'
    note: 补充实现细节并修正示例。
```

`date` 必须是可被解析的日期，`note` 必须是非空字符串。未填写时不显示变更记录区块，历史文章无需迁移。

### 从 Git 历史生成草稿

执行 `pnpm changelog:dry-run` 会扫描全部文章的逐文件 Git 历史，并生成 `docs/文章变更记录草稿.md`。该命令**不会修改文章**；它会排除目录迁移、项目重构、构建兼容修复等非文章语义提交，并根据每个提交的文章 diff 提取新增标题或生成保守摘要。若最早可见历史是迁移或重构，脚本不会错误标记为“首次发布”。

可先针对单篇文章审核：`pnpm changelog:dry-run -- --only 环境配置一览`。审核草稿后，将确认的条目手动写入对应文章的 `changelog` frontmatter；低置信条目不应直接写入。

## 文章关系与知识图谱

- 上一篇/下一篇：仅统计非草稿文章，按 `pubDate` 升序排列。
- `related`：可选文章 ID 数组，用于定义显式阅读路径；文章 ID 即内容集合中的相对路径，例如 `2026/异步生成器`。
- `series`：可选系列名称；相同名称的文章会构成系列关联。
- 同主题文章：排除当前文章，使用共同标签建立关联，最多展示 3 篇。
- 关联优先级为 `related` → `series` → `tags`。同一对文章只保留优先级最高的关系。
- 图谱和文章页均排除草稿。未填写 `related` 或 `series` 的历史文章仍可通过标签加入图谱，无需批量迁移。

```yaml
tags: ["JavaScript", "笔记"]
series: "JavaScript 异步"
related:
  - "2026/异步生成器"
```
