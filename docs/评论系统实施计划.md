# 评论系统实施计划

## 1. 目标

为当前 Astro 静态博客实现自研评论后端。第一阶段交付一个带审核、防刷、基础管理能力的最小可用评论系统。

## 2. 成功标准

- 博客文章页可以展示已审核评论。
- 访客可以提交评论，提交后进入审核队列。
- 管理员可以登录并审核、拒绝、删除评论。
- 评论提交接入 Turnstile 和基础限流。
- 评论内容按纯文本渲染，不能执行 HTML/脚本。
- API 有单元测试和关键集成测试。
- `pnpm build` 通过。
- 部署文档包含环境变量、数据库迁移、前端配置说明。

## 3. 约束

- 当前博客是 Astro SSG，不应因为评论功能改成全站 SSR。
- 评论 API 独立部署，博客前端通过 `PUBLIC_COMMENT_API_URL` 调用。
- 第一版不做用户注册系统。
- 第一版不开放评论直接公开，默认需要审核。
- 不存储明文 IP，邮箱也不在公开 API 返回。
- 修改 React 组件时优先使用现有 shadcn/ui 组件；若缺少基础组件，先补齐。

## 4. 实施切片

### 切片 1：文档和 workspace 准备

任务：
- 调整 `pnpm-workspace.yaml`，加入 `apps/*` 和 `packages/*`。
- 创建 `apps/comment-api` 基础目录。
- 明确 API、数据表、环境变量和部署方式。

测试：
- 执行 `pnpm install --lockfile-only` 或等价命令确认 workspace 配置可解析。

交付：
- `pnpm-workspace.yaml`
- `apps/comment-api/package.json`
- `apps/comment-api/tsconfig.json`

### 切片 2：后端基础服务

任务：
- 使用 Hono 初始化 API。
- 增加 `/health` 健康检查。
- 增加统一错误响应。
- 增加环境变量读取和校验。

测试：
- 单元测试环境变量校验。
- 集成测试 `/health` 返回 200。

交付：
- `apps/comment-api/src/index.ts`
- `apps/comment-api/src/config.ts`
- `apps/comment-api/src/http/errors.ts`

### 切片 3：数据库和迁移

任务：
- 接入 Drizzle ORM。
- 定义 `comments` 表。
- 生成并提交迁移文件。
- 增加数据库连接模块。

测试：
- 单元测试 schema 类型和状态枚举。
- 集成测试迁移后的评论插入和查询。

交付：
- `apps/comment-api/src/db/schema.ts`
- `apps/comment-api/src/db/client.ts`
- `apps/comment-api/drizzle/*`

### 切片 4：公开评论 API

任务：
- 实现 `GET /api/comments?postSlug=xxx`。
- 实现 `POST /api/comments`。
- 做 Zod 请求校验。
- 邮箱、IP、User-Agent 存 hash。
- 评论默认写入 `pending`。

测试：
- 评论列表只返回 `approved`。
- 提交评论会写入 `pending`。
- 无效邮箱、超长正文返回 400。
- HTML 内容作为文本保存和返回，不执行。

交付：
- `apps/comment-api/src/routes/comments.ts`
- `apps/comment-api/src/services/commentService.ts`
- `apps/comment-api/src/validation/commentSchemas.ts`

### 切片 5：防刷、CORS 和安全中间件

任务：
- 接入 Cloudflare Turnstile 服务端校验。
- 增加 CORS allowlist。
- 增加基础限流。
- 增加安全响应头。

测试：
- Turnstile 失败时 `POST /api/comments` 返回 403。
- 非允许 Origin 写请求被拒绝。
- 超过限流阈值返回 429。

交付：
- `apps/comment-api/src/services/turnstileService.ts`
- `apps/comment-api/src/middleware/cors.ts`
- `apps/comment-api/src/middleware/rateLimit.ts`

### 切片 6：管理 API

任务：
- 实现管理员登录。
- 实现待审核评论列表。
- 实现评论状态更新。
- 实现软删除。

测试：
- 未登录访问管理接口返回 401。
- 登录成功后可查看 pending 评论。
- 状态流转只允许合法值。
- 删除接口将状态改为 `deleted`。

交付：
- `apps/comment-api/src/routes/admin.ts`
- `apps/comment-api/src/services/authService.ts`
- `apps/comment-api/src/middleware/auth.ts`

### 切片 7：博客前端评论区

任务：
- 新增 `src/features/comments`。
- 实现评论列表、提交表单、loading/error/empty 状态。
- 接入 Turnstile site key。
- 修改 `src/pages/blog/[...slug].astro`，传递 `post.id`。
- 修改 `src/layouts/BlogPost.astro`，在正文后挂载评论组件。
- 修改 `src/content.config.ts`，增加 `comments` frontmatter 开关。

测试：
- 组件空状态渲染正确。
- 提交成功后显示“评论已提交，审核后展示”。
- API 错误时显示中文错误状态。
- `comments: false` 的文章不渲染评论区。

交付：
- `src/features/comments/components/CommentsSection.tsx`
- `src/features/comments/api.ts`
- `src/features/comments/types.ts`
- `src/layouts/BlogPost.astro`
- `src/pages/blog/[...slug].astro`
- `src/content.config.ts`

### 切片 8：管理端 UI

任务：
- 第一版可在博客项目内增加隐藏管理路由，或单独创建 `apps/comment-admin`。
- 实现登录页。
- 实现待审核列表。
- 实现批准、拒绝、删除操作。

测试：
- 登录失败显示中文错误。
- 审核操作成功后列表刷新。
- 未登录状态不能看到评论数据。

交付：
- 若内置在博客：`src/pages/admin/comments.astro` 和相关 React 组件。
- 若独立应用：`apps/comment-admin/*`。

### 切片 9：部署和文档回填

任务：
- 确认 API 部署平台：Cloudflare Workers、Vercel Functions 或 VPS。
- 配置数据库和迁移命令。
- 配置前端环境变量。
- 补充 README 或独立部署文档。

测试：
- 生产环境 API `/health` 可访问。
- 正式博客域名可以读取和提交评论。
- 未知 Origin 无法提交评论。
- 管理员可以审核生产评论。

交付：
- `docs/评论系统部署.md`
- 更新 `README.md` 或新增评论系统链接。

## 5. 建议优先级

第一阶段必须完成：
1. 切片 1 到 7。
2. 管理 API。
3. 最小管理 UI 或临时管理脚本。

第二阶段再做：
1. 更完整的管理端 UI。
2. 邮件通知。
3. 回复功能。
4. 审计日志。
5. 评论导出和迁移工具。

## 6. 预计工期

| 范围 | 预计时间 |
|---|---:|
| 极简 API + 前端展示 + 手动数据库审核 | 2 到 3 天 |
| API + 前端 + 管理 API + 简单管理 UI | 4 到 6 天 |
| 带通知、审计、回复和完整管理体验 | 1 到 2 周 |

## 7. 交付前验证清单

- `pnpm build`
- `pnpm --filter comment-api test`
- API 集成测试通过。
- 本地博客文章页能展示评论区。
- 提交评论进入 `pending`。
- 管理员批准后评论变为公开。
- Turnstile 失败、CORS 失败、限流触发均有明确响应。
- 文档已回填环境变量和部署步骤。
